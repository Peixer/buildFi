# Marketplace

## Purpose
The marketplace is the front door of BuilderOS. It allows investors to discover, compare, and evaluate construction projects seeking funding.

## Core Responsibilities
- Let builders submit projects
- Surface projects to investors
- Present financing opportunities clearly
- Show enough project context to support investment decisions
- Connect discovery to funding action

## Listing Data Model
Each listing should communicate:
- Project name
- Location
- Builder name
- Short investment thesis
- Funding target
- Financing structure
- Stage of construction
- Estimated returns
- Timeline
- Key milestones
- Risk level or trust indicators

## Builder Flow
1. Builder creates a profile
2. Builder do KYB process
3. Builder submits project data
4. Builder defines funding need and milestone plan
5. Project is published in the marketplace

## Investor Flow
1. Investor browses listings
2. Investor filters by project type, region, return profile, or stage
3. Investor opens detail page
4. Investor reviews milestones, financing terms, and builder credibility
5. Investor do KYC process
6. Investor proceeds to commit capital
7. Sign investment deal

## MVP Scope
- Project listing page
- Project card grid
- Project detail page
- Basic filtering
- Invest CTA
- Funding progress indicator

## Product Goals
The marketplace should reduce friction in two ways:
- make builders legible
- make opportunities comparable

## UX Principles
- Clear project summaries
- Strong trust cues
- Clean progression from browsing to investing
- Visual emphasis on stage, milestone plan, and financing structure

## Future Extensions
- Watchlists
- Personalized investor recommendations
- Reputation-weighted search ranking
- Secondary market entry point for token resale
