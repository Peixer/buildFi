# Project Tracking

## Purpose
Project Tracking gives investors ongoing visibility into how a construction project is progressing after funding. It converts a static listing into a living operational timeline.

## Core Responsibilities
- Define milestones
- Track project progress
- Show budget usage
- Report timeline updates
- Attach supporting evidence
- Build an execution history for each project

## What Needs To Be Tracked
- Construction stage
- Milestone status
- Percent completion
- Timeline changes
- Budget consumption
- Photos, reports, or proofs of work
- Delays, blockers, and risk notes

## Recommended Milestone Statuses
- Planned
- In Progress
- Submitted for Review
- Approved
- Delayed
- Completed

## Builder Flow
1. Builder defines milestones at project setup
2. Builder updates progress over time
3. Builder submits a milestone for review
4. Builder attaches proof or context
5. Status updates become visible to investors

## Investor Flow
1. Investor opens project timeline
2. Investor reviews completed and pending milestones
3. Investor sees recent updates and budget usage
4. Investor understands whether the project is on track

## MVP Scope
- Timeline or milestone view
- Status per milestone
- Project update feed
- Simple budget or spend indicator
- Milestone review state

## Product Value
This module is the trust engine after the investment has been made.
It reduces the black-box effect that normally exists in private real estate development.

## Demo Narrative
A project moves from foundation to structure. The builder updates progress, submits the milestone, and the investor sees the change reflected in the timeline and funding state.
